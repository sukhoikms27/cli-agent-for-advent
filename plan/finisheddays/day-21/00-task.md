# 00 — Задание и контекст (day-21)

## Задание курса (day-21.md)

> Индексация документов: набор документов (README/статьи/код/pdf → текст), 20-30+ страниц.
> Пайплайн: chunking → генерация эмбеддингов → сохранение индекса (FAISS/SQLite/JSON).
> Усиление: метаданные к каждому чанку (source, title/file, section, chunk_id); минимум 2
> стратегии chunking (по фиксированному размеру и по структуре) + их сравнение.

## Контекст лекции недели 5 (`05_RAG_эмбеддинги…`)

- **RAG** — дополнение ответов LLM внутренними знаниями во время инференса (≠ MCP = внешние
  источники, ≠ fine-tuning = веса). Подгрузка данных в контекст, а не вызов внешнего tool'а.
- **Эмбеддинги** — текст → вектор (768/1024 чисел); косинусное сходство (1 идентичны, 0 о разном,
  −1 противоположны) лежит в основе семантического поиска.
- **Чанкинг с overlap** (1-500, 451-950, 901-1400) решает потерю контекста на границах. Размер
  чанка 500-1000 токенов — экспериментально.
- **Ollama** — рекомендуемый инструмент для локальных эмбеддингов: `nomic-embed-text`, бесплатно,
  приватно, совместимо с облаком. Эндпоинт `POST localhost:11434/api/embed`.
- **Реранкинг** (CrossEncoder) — упомянут как «задание на неделе», но НЕ в day-21/22 stubs.

## Решения (см. план, одобрен пользователем)

| Развилка | Решение | Обоснование |
|---|---|---|
| Эмбеддинги | **Ollama локально** (`nomic-embed-text`, 768 dim) | Как в лекции; `EmbeddingClient` interface готов к облаку |
| Корпус | **Документация проекта** (MD + `.kt`) | Самореференциально; 30+ страниц без сторонних данных |
| Индекс | **JSON** (не FAISS/SQLite) | Задание разрешает; для CLI малый корпус → linear scan достаточен, 0 зависимостей |
| Чанкинг | **2 стратегии** (fixed + structural) | Требование задания; сравнение = статистика + пробный retrieval |
