## Ключевые инварианты

- **Документы не противоречат коду** — главное правило. После R1–R3 ни один документ Day 18 не должен
  упоминать `CLI_AGENT_WEATHER_CITIES`/`_INTERVAL_SECONDS` как активную конфигурацию (только в
  superseded-файлах с пометкой «устарело»).
- **Superseded-файлы не удаляются** — оставлены как историческая справка (как было изначально
  спроектировано → почему переработано). Пометка в начале — однозначный сигнал.
- **R0-redesign-note.md — единая точка правды** redesign'а: на него ссылаются все обновлённые документы.
- **Демо-сценарии** отражают pull-модель: подписка → фоновый сбор → результат через `get_weather_summary`.
