# Задача 07. Верификация

## Цель
Подтвердить, что разные профили дают разные ответы, профиль применяется автоматически и переживает рестарт.

## Manual REPL
1. `/profile set style concise` + `/profile set about "backend dev, Ktor"` + `/profile add constraint "no RxJava"` → `/profile show`. Спросить «напиши сервис авторизации» → ответ краткий, на Ktor, без RxJava (профиль учтён).
2. `/profile clear` → `/profile set style verbose` → тот же запрос → другой стиль (верификация «ответы для разных профилей»).
3. `/profile extract` после диалога о предпочтениях → профиль наполнен inferred-полями («ассистент учитывает автоматически»).
4. `cli-agent chat --auto-profile` → после 5 ходов профиль обновляется без `/profile extract`.
5. Рестарт: `cli-agent chat -c new` → новый чат всё ещё применяет профиль (global long-term).

## Автоматизация
- `./gradlew test` — новые тесты + Day 11 зелёные.
- `./gradlew build`.

## Критерии готовности (соответствие заданию)
- ✅ Профиль пользователя создан (`UserProfile`).
- ✅ Описаны предпочтения (style/format/about/constraints).
- ✅ Подключён к каждому запросу (PromptBuilder).
- ✅ Проверены ответы для разных профилей (шаг 2).
- ✅ Проверено, что ассистент учитывает автоматически (шаги 3–4).
