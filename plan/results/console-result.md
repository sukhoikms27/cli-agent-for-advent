─────────────────────────────────────────────────────────────────── Шаг 1: Определить структуру состояния (State) ───────────────────────────────────────────────────────────────────



                                                                                       Решение                                                                                       


В MVI-архитектуре состояние должно быть неизменяемым (immutable). Поэтому используем data class с List<Boolean> (а не BooleanArray, который не переопределяет equals/hashCode
корректно для data class).

╭────────────────────────────────────────────────────╮
│data class LockerState(                             │
│    val lockers: List<Boolean> = List(100) { false }│
│)                                                   │
╰────────────────────────────────────────────────────╯


                                                                                      Пояснение                                                                                      


• List<Boolean> — иммутабельная коллекция, что соответствует принципам MVI (State — snapshot, не мутируется на месте).
• List(100) { false } — фабричная функция создаёт список из 100 элементов, все false (закрыты).
• Индекс 0 ↔ шкафчик №1, индекс 99 ↔ шкафчик №100.
• Значение false = закрыт, true = открыт.
• Начальное состояние — все 100 шкафчиков закрыты, что соответствует условию задачи.

──────────────────────────────────────────────────────────────── Шаг 2: Организовать внешний цикл (симуляция людей) ─────────────────────────────────────────────────────────────────



                                                                                       Решение

│for (person in 1..100) {                  │
│    // здесь будет внутренний цикл (шаг 3)│
│}                                         │
╰──────────────────────────────────────────╯


                                                                                      Пояснение                                                                                      


• 1..100 — диапазон в Kotlin, включающий обе границы (от 1 до 100 включительно).
• Переменная person последовательно принимает значения 1, 2, 3, … 100 — номер человека.
• На каждой итерации один «человек» обходит шкафчики, кратные его номеру.
• Тело цикла пока пустое — внутренний цикл будет добавлен на следующем шаге.
• 100 итераций = 100 человек, что точно соответствует условию задачи.

─────────────────────────────────────────────────────── Шаг 3: Организовать внутренний цикл (симуляция прохода по шкафчикам) ────────────────────────────────────────────────────────



                                                                                       Решение                                                                                       


╭─────────────────────────────────────────────────╮
│for (person in 1..100) {                         │
│    for (locker in person..100 step person) {    │
│        // здесь будет инверсия состояния (шаг 4)│
│    }                                            │
│}                                                │
╰─────────────────────────────────────────────────╯


                                                                                      Пояснение                                                                                      


• person..100 step person — диапазон от person до 100 с шагом person.
• Например, для person = 3 переменная locker примет значения: 3, 6, 9, 12, …, 99.
• Для person = 1 — все шкафчики от 1 до 100 (человек №1 открывает всё).
• Для person = 100 — только значение 100 (один шкафчик).
• Условие locker ≤ 100 обеспечивается верхней границей диапазона ..100.
• Ключевое слово step — идиоматичный способ задания шага в диапазонах Kotlin; альтернатива через while была бы многословнее.

──────────────────────────────────────────────────────────────────────── Шаг 4: Изменить состояние (Toggle) ─────────────────────────────────────────────────────────────────────────


Решение:

В теле внутреннего цикла инвертируем состояние шкафчика:

╭────────────────────────────────────────────────────────╮
│workingLockers[locker - 1] = !workingLockers[locker - 1]│
╰────────────────────────────────────────────────────────╯

В контексте уже созданных циклов:

╭────────────────────────────────────────────────────────────────╮
│for (person in 1..100) {                                        │
│    for (locker in person..100 step person) {                   │
│        workingLockers[locker - 1] = !workingLockers[locker - 1]│
│    }                                                           │
│}                                                               │
╰────────────────────────────────────────────────────────────────╯

Пояснение:

• Используем локальную изменяемую рабочую копию (BooleanArray или MutableList<Boolean>) внутри Reducer, чтобы эффективно выполнять симуляцию.
• locker - 1 — потому что номера шкафчиков начинаются с 1, а индексы массива/списка в Kotlin — с 0.
• Оператор ! меняет значение на противоположное: false → true (закрыт → открыт), true → false (открыт → закрыт).
• В MVI итоговое состояние будет создано как неизменяемое позже, после завершения всех переключений.

│                                                                          │
│// Reducer                                                                │
│fun reduce(state: LockersState, intent: LockersIntent): LockersState =    │

│        is LockersIntent.CalculationCompleted -> state.copy(              │
│            isLoading = false,                                            │
│            openLockers = intent.openLockers,                             │

Пояснение:

• Intent — CalculationCompleted — иммутабельный объект с результатами вычислений.
• State — LockersState — data class с полями для списка и количества.
