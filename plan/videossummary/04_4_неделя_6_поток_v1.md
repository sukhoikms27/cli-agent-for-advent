## Слайд 15

**Заголовок/тема:** Зависимости / SDK


### Текст и визуальная информация со слайда
- Слайд с кодом настройки зависимостей.
- Kotlin/Gradle: переменные версий `mcpVersion`, `slf4jVersion`, `ktorVersion`.
- Зависимости: `io.modelcontextprotocol:kotlin-sdk`, `slf4j-nop`, Ktor client content negotiation, Kotlinx JSON serialization.
- Ниже команды для Node.js:

```bash
npm install @modelcontextprotocol/sdk zod@3
npm install -D @types/node typescript
```

- Слайд с Kotlin-кодом создания сервера.
- Видны элементы вроде `Server(...)`, `ServerOptions(...)`.
- Transport: `StdioServerTransport(system.in.asSource().buffered(), system.out.asSink().buffered())`.
- Смысл: MCP-сервер подключается через stdio transport.

- Kotlin-код с `@Serializable` data class.
- Классы: `Points`, `Properties`.
- Функция `HttpClient.getForecast(latitude: Double, longitude: Double): List<String>`.
- Внутри: запрос `/points/$latitude,$longitude`, затем запрос прогноза по `points.properties.forecast`.
- Возвращаются строки с name, temperature, wind speed/direction и detailed forecast.

- Kotlin-код настройки `HttpClient`.
- `defaultRequest` использует URL `https://api.weather.gov`.
- В headers видны `Accept: application/geo+json` и `User-Agent`.
